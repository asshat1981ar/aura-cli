# Track Plan: Recursive Self-Improvement (RSI) Lifecycle

## Phase 1: Self-Directed Debt Reduction
- [ ] Task: Enable Architectural Signal Discovery
    - [ ] Implement: Verify `AutonomousDiscovery` correctly enqueues goals for circular dependencies and hotspots.
    - [ ] Test: Run a discovery scan on a known "messy" state and verify goals appear in `aura goal status`.
- [ ] Task: Automated Refactoring Loop
    - [ ] Implement: Ensure `EvolutionLoop` can successfully process a "refactor hotspot" goal generated by discovery.
    - [ ] Verify: A hotspot's complexity score should decrease after a successful RSI cycle.

## Phase 2: Learning System Optimization
- [ ] Task: Meta-Strategy Adaptation
    - [ ] Implement: `AdaptivePipeline` should periodically review its own win/loss records and adjust the `min_score` or `top_k` parameters in `aura.config.json`.
    - [ ] Test: Simulate high failure rates for a goal type and verify that AURA switches to "deep" intensity automatically.
- [ ] Task: Router Performance Tuning
    - [ ] Implement: `RouterAgent` should automatically bench models that consistently underperform in latency or success rate beyond a certain threshold.

## Phase 3: Skill Synthesis & Evolution
- [ ] Task: Autonomous Skill Creation
    - [ ] Implement: A special `skill_generator` agent/skill that can write and register new Python skills based on recurring task patterns.
    - [ ] Verify: AURA identifies a missing capability (e.g., "YAML linting") and creates a basic `YamlLinterSkill`.
- [ ] Task: System Evolution Verification
    - [ ] Final Integration: Run AURA in a fully autonomous "evolve" mode for 50 cycles and audit the architectural delta.
