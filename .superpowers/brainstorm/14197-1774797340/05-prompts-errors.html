Sections 3 & 4 of the fleet prompt design

§3 — LLM Prompt Templates
classify-issue-v1
You are a GitHub issue classifier for an autonomous dev fleet. Given the issue title and body, return JSON: { "type": "bug|feature|refactor|security|docs", "confidence": 0.0-1.0, "summary": "one sentence", "key_files": ["path/hint"], "complexity": "low|medium|high" } Only return valid JSON. No explanation.
→ routes WF-0 Switch node
bug-analyst-v1
You are a senior engineer doing root cause analysis. Context: {aura_skills_output} Issue: {issue_body} Return a patch strategy: - root_cause: what went wrong and why - affected_files: list with line ranges - fix_approach: step-by-step - regression_test: what to add
→ injected as context into AURA /run goal
feature-planner-v1 / refactor-planner-v1
You are an architect. Given the AURA skills analysis and the issue spec, produce an implementation plan: - components_to_create: [] - components_to_modify: [] - interfaces: API contracts - test_strategy: unit + integration coverage - risks: []
→ injected as context into AURA /run goal
security-analyst-v1
You are a security engineer. Given the scanner output: {security_scanner_findings} Triage each finding: - severity: critical|high|medium|low - cve_id: if known - patch: minimal safe fix - verify_with: what SAST check confirms the fix Priority: fix critical/high only. Flag medium/low.
→ injected as context into AURA /run goal
§4 — Retry + Escalation Logic
1
Quality gate check — before PR creation, generation_quality_checker scores the output. Score <70 triggers an immediate retry without counting against the 3-attempt budget (quality retries are free, up to 2).
2
CI failure retry — if CI checks fail after PR push, the sub-flow re-runs with the CI error injected as additional context into the LLM analyst prompt. retryCount++ on each attempt.

Retry enrichment: previous_attempt_error field added to all prompts so the LLM knows what failed.
3
Max retries reached (retryCount = 3) — fleet posts a structured blocker comment to the issue:

🚫 Fleet blocked after 3 attempts.
Last error: {ci_error}
Skills report: {link}
Attempted branches: {list}
Label: fleet:blocked

Labels applied: fleet:blocked replaces fleet:processing
4
Success close — on merge: removes fleet:processing, adds fleet:done, posts summary comment with PR link + quality score, closes issue.
