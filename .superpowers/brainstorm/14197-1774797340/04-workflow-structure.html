Fleet Workflows — Internal Node Structure

6 n8n workflows. Each box = a node. Colours = node type.

🎛️ WF-0: Master Dispatcher
TRIGGERGitHub Webhook — issue.opened, issue.labeled events
↓
LLMIssue Classifier — reads title+body, returns {type, confidence, summary, key_files[]}prompt: classify-issue-v1
↓
CODERetry State Init — sets retryCount=0, stamps fleetRunId, adds label fleet:processing to issue
↓
SWITCHType Router — routes to WF-1 through WF-5 based on classifier output. Fallback → Feature Flow
🐛 WF-1: Bug-Fix Flow
HTTPAURA Skills fan-out error_pattern_matcher, type_checker, test_coverage_analyzer
↓
LLMBug Analyst — root cause + patch strategyprompt: bug-analyst-v1
↓
HTTPAURA Loop /run — goal: "Fix bug: {summary}" with patch context injected
↓
SUB→ WF-5 Merge
✨ WF-2: Feature Flow
HTTPAURA Skills fan-out symbol_indexer, architecture_validator, api_contract_validator
↓
LLMFeature Planner — spec extraction + scaffold planprompt: feature-planner-v1
↓
HTTPAURA Loop /run — goal: "Implement: {spec}"
↓
SUB→ WF-5 Merge
♻️ WF-3: Refactor Flow
HTTPAURA Skills fan-out complexity_scorer, code_clone_detector, tech_debt_quantifier, refactoring_advisor
↓
LLMRefactor Planner — decomposition plan, preserves test coverageprompt: refactor-planner-v1
↓
HTTPAURA Loop /run — goal: "Refactor: {target}" with debt context
↓
SUB→ WF-5 Merge
🔒 WF-4: Security Flow
HTTPAURA Skills fan-out security_scanner, dependency_analyzer, linter_enforcer
↓
LLMSecurity Analyst — CVE triage, patch priorityprompt: security-analyst-v1
↓
HTTPAURA Loop /run — goal: "Fix security: {finding}"
↓
SUB→ WF-5 Merge
📄 WF-5a: Docs Flow
HTTPAURA Skills fan-out symbol_indexer, doc_generator
↓
LLMDoc Writer — generates docstrings, README sectionsprompt: doc-writer-v1
↓
HTTPAURA Loop /run — goal: "Document: {scope}"
↓
SUB→ WF-5b Merge
✅ WF-6: Push PR + Auto-Merge (shared convergence)
HTTPgeneration_quality_checker — scores output before PR. Score <70 → retry signal
↓
GHgh pr create — branch: fleet/{type}/{issue-number}, body includes AURA skill report + quality score
↓
WAITCI Poll — waits for status checks to pass (max 10 min)
↓
SWITCHCI Result → pass: gh pr merge --squash · fail + retryCount<3: back to sub-flow · fail + retryCount=3: blocker comment + label fleet:blocked
↓
GHIssue Close — posts summary comment, removes fleet:processing, closes issue
